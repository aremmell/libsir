Detailed Description

Compile-time configuration that can be modified to alter the behavior of libsir.

Attention
Don't make changes to anything until you've studied how these values are used elsewhere in the source code.

◆ SIR_ERRORFORMAT

#define SIR_ERRORFORMAT   "Error in %s (%s:%u): '%s'"

#include <sirconfig.h>

The format string for error messages returned by _sir_geterror.

  • The first %s format specifier is the function name.
  • The second %s is the file name.
  • The %lu is the line number in the file.
  • The third %s is the error message.
Remarks
sample: ‘Error in findneedle (haystack.c:384): 'Too much hay’`

  • The first %s format specifier is the message (e.g., SIR_FHBEGIN, SIR_FHROLLED)
  • The second %s is the current date/time (see SIR_FHTIMEFORMAT).

◆ SIR_FNAMEFORMAT

#define SIR_FNAMEFORMAT   "%s-%s%s"

#include <sirconfig.h>

The format string for rolled/archived log file names.

  • The first %s format specifier is the name part (the name up to the last '.') of the original file name.
  • The second %s is the time stamp as defined by SIR_ROLLTIMEFORMAT.
  • The third %s is the extension part (the name after and including the last '.') of the original file name if one is present.
Remarks
sample: oldname-18-05-05-122049.log
